# Integration Contract (UNMAPPED Backend v1)

This contract is for frontend integration from Lovable. Endpoints are live under `/api/*` and return mock-backed responses by default.

## Runtime behavior

- `USE_MOCK_API=true` (default): endpoints return static JSON from `public/mock/*` and `data/country-configs/indonesia.json`.
- `USE_MOCK_API=false`: selected endpoints can call Claude for generated responses.
- Claude key is backend-only and read only from `process.env.ANTHROPIC_API_KEY`.

## Endpoints

### `GET /api/demo/user`

Purpose: return the current user profile.

Request body: none.

Response shape:

```json
{
  "userId": "string",
  "countryCode": "string",
  "provinceCode": "string",
  "displayName": "string",
  "age": 0,
  "educationLevel": "string",
  "employmentStatus": "string",
  "skills": ["string"],
  "preferredSectors": ["string"]
}
```

### `POST /api/skills/extract`

Purpose: extract normalized skills from transcript text.

Request shape:

```json
{
  "transcript": "string"
}
```

Response shape:

```json
{
  "extractedSkills": [
    {
      "name": "string",
      "confidence": 0
    }
  ],
  "missingCriticalSkills": ["string"]
}
```

### `POST /api/risk/score`

Purpose: generate risk score, risk drivers, and recommended pathways.

Request shape:

```json
{
  "skills": [
    {
      "name": "string",
      "confidence": 0
    }
  ]
}
```

Response shape:

```json
{
  "userId": "string",
  "overallRiskScore": 0,
  "riskLevel": "low|medium|high",
  "drivers": [
    {
      "factor": "string",
      "weight": 0,
      "status": "string"
    }
  ],
  "recommendedPathways": [
    {
      "pathwayId": "string",
      "title": "string",
      "priority": 0
    }
  ]
}
```

### `POST /api/pathways/simulate`

Request shape:

{
  "skills": ["any"],

  "drivers": ["optional array"]

```json

  "userId": "string",
  "recommendedPathways": [
    {
      "pathwayId": "string",
      "title": "string",
      "priority": 0
    }
  ]
}
```

### `GET /api/data/worldbank/:countryCode`

Purpose: return the committed World Bank WDI extract for a country (ISO3 code used by the World Bank API, for example `IDN`, `GHA`, `IND`). The demo server reads static JSON generated by `npm run fetch:worldbank` and does not call the World Bank API at runtime.

Path parameter:

- `countryCode`: ISO3 country code (case-insensitive), must match a file under `data/worldbank/`.

Response shape:

```json
{
  "country": "Indonesia",
  "country_code": "IDN",
  "indicators": {
    "employment_services_pct": {
      "2024": 0,
      "2023": 0
    },
    "employment_agriculture_pct": {},
    "employment_industry_pct": {},
    "unemployment_rate": {},
    "youth_unemployment_rate": {},
    "youth_neet_rate": {},
    "gdp_per_capita_usd": {},
    "internet_users_pct": {},
    "mobile_cellular_subscriptions_per_100": {},
    "human_capital_index": {},
    "labor_force_participation_rate": {},
    "informal_employment_pct": {}
  },
  "fetched_at": "ISO-8601 string",
  "source": "World Bank WDI API",
  "indicator_codes": {
    "employment_services_pct": "SL.SRV.EMPL.ZS"
  }
}
```

Notes:

- Each numeric value in `indicators.*` maps to the WDI code listed under `indicator_codes` for that key.
- Years with `null` from the upstream API are omitted; clients should use the most recent year key present per series.
