# Integration Contract (UNMAPPED Backend v1)

This contract is for frontend integration from Lovable. Endpoints are live under `/api/*` and return mock-backed responses by default.

## Runtime behavior

- `USE_MOCK_API=true` (default): endpoints return static JSON from `public/mock/*`, `data/country-configs/indonesia.json`, and other committed files under `data/*` (no upstream API calls at request time).
- `USE_MOCK_API=false`: selected endpoints can call Claude for generated responses.
- Claude key is backend-only and read only from `process.env.ANTHROPIC_API_KEY`.

## Endpoints

### Canonical vs compatibility routes

- Canonical (person2) contracts are:
  - `GET /api/demo/user`
  - `GET /api/dashboard/province-risk`
  - `GET /api/dashboard/interventions` and `POST /api/dashboard/interventions`
  - `POST /api/skills/extract`
  - `POST /api/risk/score`
  - `POST /api/data-intake/analyze`
- Compatibility aliases are exposed for Lovable/mobile flows:
  - `GET /api/profile`
  - `GET /api/passport`
  - `GET /api/dashboard/provinces`
  - `GET /api/dashboard/interventions-legacy`

### `GET /api/data/worldbank/:countryCode`

Purpose: return the committed World Bank WDI extract for a country (ISO3 code used by the World Bank API, for example `IDN`). The demo server reads static JSON generated by `npm run fetch:worldbank` and does not call the World Bank API at runtime. The `available` field in 404 responses lists whichever `*.json` files exist under `data/worldbank/`.

Path parameter:

- `countryCode`: ISO3 country code (case-insensitive), must match a file under `data/worldbank/`.

Response shape:

```json
{
  "country": "Indonesia",
  "country_code": "IDN",
  "indicators": {
    "employment_services_pct": {
      "2024": 0,
      "2023": 0
    },
    "employment_agriculture_pct": {},
    "employment_industry_pct": {},
    "unemployment_rate": {},
    "youth_unemployment_rate": {},
    "youth_neet_rate": {},
    "gdp_per_capita_usd": {},
    "internet_users_pct": {},
    "mobile_cellular_subscriptions_per_100": {},
    "human_capital_index": {},
    "labor_force_participation_rate": {},
    "informal_employment_pct": {}
  },
  "fetched_at": "ISO-8601 string",
  "source": "World Bank WDI API",
  "indicator_codes": {
    "employment_services_pct": "SL.SRV.EMPL.ZS"
  }
}
```

Notes:

- Each numeric value in `indicators.*` maps to the WDI code listed under `indicator_codes` for that key.
- Years with `null` from the upstream API are omitted; clients should use the most recent year key present per series.

Purpose: return committed Wittgenstein Centre WIC education projections (SSP2) derived from the batch `pop-age-edattain` extract (run `npm run fetch:wittgenstein` to refresh).

### `POST /api/data-intake/analyze`

Purpose: operator document intake. In **mock** mode with `rawText` containing map keywords (`Sulawesi`, `Jakarta` / `DKI`, `Nusa Tenggara`, `NTT`), returns **map-ready** province labor data from `public/mock/dataIngestionMapResponse.json`. With **`USE_MOCK_API=false`** and a non-empty `rawText`, the server calls Claude with a multi-province labor-report parser; on parse/validation failure it falls back to the same static map mock (HTTP 200, never a hard error for the dashboard).

Request shape (typical):

```json
{
  "fileName": "optional.pdf",
  "rawText": "pasted government report text",
  "textPreview": "optional"
}
```

**Map ingest response** (fields merged at the top level with `analysisId`, `model`, `mockRouting`, `proposedUpdates`, `proposedCount`, `message`):

```json
{
  "analysisId": "uuid",
  "fileName": "",
  "rawTextReceived": true,
  "mockRouting": {
    "scenario": "labor_market_map",
    "routing": "rawText_map_keywords | claude_labor_map_parser | mock_fallback_after_claude",
    "detail": "string"
  },
  "model": "mock | claude | mock_fallback | mock_emergency",
  "reportType": "laborMarket",
  "detectedCountry": "Indonesia",
  "detectedLanguage": "id",
  "reportTitle": "string",
  "provincesFound": 3,
  "provinces": [
    {
      "name": "Sulawesi Selatan",
      "nameStandardized": "Sulawesi Selatan",
      "totalEmployed": 4210000,
      "sectors": {
        "agriculture": { "count": 1608000, "pct": 38.2 },
        "manufacturing": { "count": 375000, "pct": 8.9 },
        "retail": { "count": 787000, "pct": 18.7 },
        "services": { "count": null, "pct": null },
        "transportation": { "count": null, "pct": null }
      },
      "unemploymentRate": 5.31,
      "avgWageMonthly": 3200000,
      "internetPenetration": null,
      "riskScore": 0.31,
      "riskLevel": "lower_risk | medium | high | severe",

        "2026": 0.09,
        "2027": 0.11,
        "2028": 0.13,
        "2029": 0.15,
```

Notes:

- **`geoMatch`**: join key for admin GeoJSON; `property` is fixed to `NAME_1`, `value` is the official province label (e.g. `DKI Jakarta`, `Nusa Tenggara Timur`).

- **`nationalBaseline.internetPenetration`**: refreshed from committed `data/worldbank/IDN.json` (`IT.NET.USER.ZS` latest) when the server enriches the payload.
- Non-map mock behavior (demo PDF filenames or keyword `rawText` for digital/labor/intervention **proposedUpdates**) is unchanged when map keywords are not present.
